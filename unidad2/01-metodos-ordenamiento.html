CARRERA: Computación ASIGNATURA: Estructura de Datos
NRO. PRÁCTICA: 1 TÍTULO PRÁCTICA: Algoritmos de Ordenamiento
OBJETIVO (Colocar el o los objetivos que se alcanzarán al desarrollar la práctica):
  • Relaciona las técnicas de ordenamiento.
  • Desarrolla módulos que emplean técnicas de ordenamiento.
INSTRUCCIONES:
  1. Generar una aplicación en Java que permita ordenar un arreglo de enteros utilizando únicamente los siguientes algoritmos de ordenamiento:
    • Inserción
    • Shell
    Cada algoritmo debe permitir ordenar de forma ascendente o descendente, según la opción seleccionada por el usuario.
  2. Funcionamiento

    Entrada

    • El arreglo ya debe estar predefinido.
    • El programa debe mostrar un menú con las opciones:
      • Ejecutar ordenamientos.
      • Salir.
    • Si el usuario selecciona ejecutar, debe elegir el tipo de orden para cada algoritmo:
      • Inserción ascendente o descendente.
      • Shell ascendente o descendente.
    • Los pasos intermedios deben mostrarse siempre.

    Desarrollo

    • El método de Inserción debe aceptar como parámetro el arreglo original y una variable booleana para definir el orden:
      • true: ascendente.
      • false: descendente.
    • El método de Shell debe aceptar como parámetro el arreglo original y una variable booleana para definir el orden:
      • true: ascendente.
      • false: descendente.
    • El resultado de cada algoritmo debe mostrar:
      • arreglo original.
      • comparaciones realizadas.
      • cambios realizados.
      • estado del arreglo después de cada cambio o comparación.
      • arreglo ordenado.
    • El arreglo a ordenar deberá ser siempre el original, por lo tanto, cada método debe trabajar con una copia independiente del arreglo.

    Salida

    • Arreglo original.
    • Pasos intermedios del método Inserción.
    • Arreglo ordenado por Inserción.
    • Número de comparaciones.
    • Número de cambios.
    • Número de iteraciones.
    • Pasos intermedios del método Shell.
    • Arreglo ordenado por Shell.
    • Número de comparaciones.
    • Número de cambios.
    • Número de iteraciones.
  3. Requerimientos
    • El código debe estar estructurado en clases con diseño orientado a objetos.
    • Cada método de ordenamiento debe retornar un arreglo de enteros con tres valores:
      • cantidad de comparaciones.
      • cantidad de cambios.
      • cantidad de iteraciones.
    • Se debe validar la entrada para evitar errores.
    • No se permite usar métodos de ordenamiento propios de Java, como Arrays.sort().
  4. Consideraciones
    • En caso de copia, tendrán calificación de cero.
    • Subir al AVAC el proyecto con su enlace del repositorio Git.
    • La tarea es individual.
    • Se pasará el desarrollo por validador de código generado por IA.
    • Se validará el tipo de entradas.
    • La opción del menú no puede aceptar letras, valores vacíos o números fuera del rango permitido.
    • Las opciones de orden deben validarse correctamente.
    • El arreglo original no debe ser modificado directamente.

ACTIVIDADES POR DESARROLLAR

1. Definición del arreglo

El estudiante debe trabajar con el siguiente arreglo predefinido:

int[] arreglo = { 12, -7, 25, 0, -15, 33, 19, -22, 5, 48, -3 };

No se debe modificar este arreglo directamente. Cada algoritmo debe trabajar con una copia.

2. Implementación de los algoritmos

El estudiante debe implementar las siguientes clases y métodos:

El menú estará en el método main.

Cada método debe:

  • Ordenar el arreglo según el valor de ascendente.
  • Mostrar todos los pasos del proceso.
  • Contabilizar:
    • comparaciones
    • cambios
    • iteraciones
